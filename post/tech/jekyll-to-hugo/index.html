1. Hello，Hugo！

之前使用 Jekyll 引擎构建的静态博客放在 GitHub Pages 已经很长一段时间没更新了，我对配置 Ruby 环境和安装 Jekyll 很没有耐心（不熟悉的技术以及网速慢），便不想多折腾。

知道 Hugo 已经很久了，最近尝试了下发现环境的搭建很简单，找了个不错的主题就重新开始了。

1.1. 安装 Hugo

参考官网的文档：Install Hugo | Hugo，选择自己的系统和可用的工具安装即可。

本文使用 Windows 系统，并已安装 Scoop 命令行安装程序。

1

scoop install hugo

验证

1
2

λ hugo version
Hugo Static Site Generator v0.68.3-157669A0 windows/amd64 BuildDate: 2020-03-24T12:04:36Z

注：如果需要将博客托管到 GitHub Pages 建议安装 Git，另外可以安装 Cmder 使用 Tab 键补全命令，比起 CMD 和 Git Bash，Cmder 用起来还是很爽的。

1
2

scoop install git
scoop install cmder

1.2. 创建 Hugo 站点

创建站点，命令格式：hugo new site [path] [flags]，下面的命令将在当前目录下创建 hugo.pysense.github.io 目录。

λ hugo new site hugo.pysense.github.io

Just a few more steps and you're ready to go:

使用命令创建站点后的目录结构如下

1
2

λ ls hugo.pysense.github.io
archetypes/  config.toml  content/  data/  layouts/  static/  themes/
  • archetypes/default.md 新建文章使用的模板
  • config.toml 配置文件
  • content/ 文章
  • static/ 静态文件
  • themes/ 主题

1.3. 添加 Hugo 主题

可以从 Hugo Themes 选择喜欢的主题，本文使用的主题为：olOwOlo/hugo-theme-even，主题的安装与配置参考所使用的主题提供的说明文档。

将主题添加到 themes 目录

1
2
3

cd hugo.pysense.github.io
git init
git submodule add https://github.com/olOwOlo/hugo-theme-even themes/even

主题提供了样例文件和配置文件，将其复制到当前目录下

1

cp -r themes/even/exampleSite/* .

1.4. 启动 Hugo 本地服务

1

hugo server -D

该命令在本机启动 Web 服务，监听在 1313 端口，浏览器访问 http://localhost:1313/ 即可查看生成的博客。

注：-D 参数将同时构建 content 目录下文件中标记 draft: true 的文章。

2. 部署到 GitHub Pages

2.1. 修改主题配置文件

根据自己的情况修改根目录下的 config.toml 配置文件，其他配置参考主题的说明文档1

因为刚接触 Hugo，所以并不想太深入地了解和自定义配置，如果有兴趣可以参考这篇文章：Hugo 从入门到会用 - olOwOlo’s Blog。

2.2. 添加博客文章

先删除 content 目录下先前使用的样例文件，然后使用以下命令创建自己的第一篇 Hugo 文章。

1

hugo new post/my-first-post.md

注：hugo 大部分主题使用 posts 目录存放博客文章，不过 even 主题使用的是 post 目录。（可以在主题配置文件中更改，同时要更新主题下的 layouts/post 目录）

使用任何编辑器修改文章内容，之后刷新浏览器即可查看效果。

如果希望在 post 目录下创建层级目录，直接在路径中添加对应的目录名即可：

1

hugo new post/tech/test.md

2.3. 创建 About 页面

About 页面是个特殊的页面，因此可用使用以下命令来创建：

1
2

λ hugo new about.md
P:\Personal\Blog\hugo.pysense.github.io\content\about.md created

为了在页眉添加 About 的链接，还需要修改配置文件，在合适的位置添加：

1
2
3
4
5

[[menu.main]]
  name = "关于" # About
  weight = 50
  identifier = "about"
  url = "/about/"

2.4. 发布到 GitHub Pages

需要说明一下，GitHub Pages 有两种类型，本博使用的是账户站点，就是以 <username>.github.io 命名的仓库，具体可以查看 GitHub Pages。

2.4.1. 手动发布到 GitHub Pages

因为站点目录本身已经有初始化 Git 仓库，为了单独上传 public 目录，我的做法是在配置文件中将 public 目录指定到站点目录之外：

1

publishdir = "../hugo.pysense.github.io.public"

通过 hugo 命令生成发布文件，之后初始化 Git 仓库并上传到 GitHub Pages 站点：

1
2
3
4
5
6
7

cd ../hugo.pysense.github.io.public
git init
git remote add upstream git@github.com:pysense/pysense.github.io
git add .
git commit -m "Initial commit"
git push --set-upstream upstream master
git push -f upstream

第一次 push 使用 -f 参数，表示强制推送，这将导致远程丢失所有的提交记录，下次再提交时就无需提供 -f 参数了。

2.4.2. 通过 Wercker 服务自动部署到 GitHub Pages

使用 Wercker 的服务自动部署到 GitHub Pages，参考：Deployment with Wercker | Hugo

初始化 Git 仓库，在站点根目录执行：

1

git init

因为 Wercker 将在稍后自动生成发布目录 ./public，因为将该目录添加到忽略文件 .gitignore：

1
2

λ cat .gitignore
/public

为了防止 Wercker 报错，还需要在静态目录下存放点文件，为了简化操作，只以添加 robots.txt 为例：

1
2
3

λ cat static/robots.txt
User-agent: *
Disallow:
